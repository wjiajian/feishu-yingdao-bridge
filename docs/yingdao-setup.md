# 影刀端实施手册

本文档只覆盖影刀应用和高级任务计划配置，不包含飞书开放平台和服务端部署。

## 一、实施目标

影刀端要完成 3 件事：

1. 创建或确认可执行的影刀应用
2. 为每个应用准备输入参数
3. 用高级任务计划创建一个 Webhook 触发入口，并把参数映射到目标影刀应用

完成后，每个业务应用都要向飞书端和服务端提供这些值：

- `app_code`
- `app_name`
- `webhook_url`
- 影刀应用输入参数列表

## 二、前置条件

开始前先确认：

- 你已经可以登录影刀控制台
- 目标影刀应用已经开发完成，或至少已有可发版主流程
- 影刀应用所需输入参数已经明确
- 目标机器人组、运行账号、授权方式已经准备好

影刀高级任务计划相关能力参考：

- 影刀高级任务计划入门  
  https://www.yingdao.com/community/detaildiscuss?id=838730599324614656

## 三、准备影刀应用

每个飞书侧可见应用，影刀侧都应有一个正式可发布的影刀应用版本。

配置步骤：

1. 打开目标影刀应用。
2. 在主流程中定义输入参数。
3. 每个参数名称要和飞书侧 `field_key` 一一对应。
4. 参数方向设置为“输入”。
5. 保存并发布应用版本。

参数命名建议：

- 使用英文或下划线命名
- 不要包含空格
- 不要频繁修改已上线参数名
- 一旦修改参数名，必须同步更新飞书多维表格里的 `field_key`

推荐做法：

- `employee_no`
- `reason`
- `start_date`
- `end_date`

## 四、创建高级任务计划

### 1. 进入创建入口

按影刀控制台路径进入：

`调度任务 -> 高级任务计划 -> 新建高级任务`

该路径在影刀官方社区文章里有明确说明：

- https://www.yingdao.com/community/detaildiscuss?id=838730599324614656

### 2. 配置触发器

1. 新建高级任务计划。
2. 触发事件选择 `Webhook 触发器`。
3. 保存后复制系统生成的 Webhook 地址。
4. 记录该地址，稍后写入飞书多维表格 `apps.webhook_url`。

官方社区对 Webhook 触发器的描述是：收到 HTTP 请求时开始执行流程，适合和已有系统结合。这个场景正好对应本项目的服务端转发模式。来源：

- https://www.yingdao.com/community/detaildiscuss?id=838730599324614656

### 3. 配置执行动作

1. 在高级任务计划里新增执行动作。
2. 选择“运行影刀 RPA”。
3. 在应用选择框里选择刚发布的目标影刀应用。
4. 配置运行所需授权、账号或机器人组。
5. 打开输入参数映射界面。

### 4. 参数映射

将 Webhook 请求体中的字段映射到影刀应用输入参数。

本项目服务端发送的是扁平 JSON，请按下面规则映射：

业务参数：

- `employee_no` -> 影刀参数 `employee_no`
- `reason` -> 影刀参数 `reason`

元信息参数：

- `_meta_request_id`
- `_meta_app_code`
- `_meta_user_open_id`
- `_meta_user_name`
- `_meta_submit_time`

如果影刀应用本身不需要这些元信息，可以不映射；如果需要日志追踪，建议至少映射 `_meta_request_id`。

服务端负载构造逻辑见：

- [src/core/yingdao-service.js](/D:/Project/feishu-shadowbot/src/core/yingdao-service.js#L3)

### 5. 发布高级任务计划

1. 参数映射完成后，点击发布。
2. 用影刀提供的测试能力先发一条模拟请求。
3. 确认高级任务计划能成功拉起目标应用。

## 五、为飞书侧准备应用清单

每个影刀应用上线前，需要整理一份清单交给飞书配置人员：

| 项 | 说明 |
|---|---|
| `app_code` | 例如 `leave_sync` |
| `app_name` | 例如“请假同步” |
| `description` | 飞书卡片说明文案 |
| `webhook_url` | 高级任务计划生成的 Webhook 地址 |
| `field_key` 列表 | 对应影刀输入参数 |
| `field_type` 列表 | 飞书卡片字段类型 |
| `required` 列表 | 是否必填 |
| `validation_regex` | 可选，字段格式约束 |
| `success_text` | 例如“已提交，稍后处理” |

## 六、影刀端联调步骤

1. 在服务端未接入飞书前，先用 Postman 或 curl 直接请求影刀 Webhook。
2. 验证高级任务计划确实能收到请求。
3. 验证参数能正确映射到影刀应用。
4. 验证影刀应用是否在正确机器人组执行。
5. 验证异常时控制台是否能看到失败原因。

建议先用下面格式做直连测试：

```json
{
  "_meta_request_id": "debug-001",
  "_meta_app_code": "leave_sync",
  "_meta_user_open_id": "ou_xxx",
  "_meta_user_name": "张三",
  "_meta_submit_time": "2026-04-07T10:20:30+08:00",
  "employee_no": "123456",
  "reason": "事假"
}
```

## 七、影刀端常见问题

### 1. 高级任务计划没被触发

重点检查：

- Webhook 地址是否复制完整
- 请求方法是否为 `POST`
- 请求体是否为 JSON
- 目标高级任务计划是否已发布

### 2. 高级任务计划触发了，但影刀应用没拿到参数

重点检查：

- 影刀应用参数是否设置为“输入”
- 参数名是否与飞书侧 `field_key` 一致
- 高级任务计划里是否完成了字段映射

### 3. 影刀应用执行失败

重点检查：

- 运行授权是否有效
- 机器人组是否有空闲机器人
- 影刀应用是否已发布最新版本
- 输入参数是否缺必填值

## 八、与本项目的边界关系

当前实现里，影刀端被视为一个稳定的 Webhook 接收方。服务端不会替影刀补业务逻辑，只做：

1. 权限校验
2. 表单参数收集
3. 请求体组装
4. Webhook 转发

因此影刀端必须自行保证：

- 高级任务计划已发布
- 输入参数定义正确
- 执行动作与授权可用
