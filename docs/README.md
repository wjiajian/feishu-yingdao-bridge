# 实施文档总览

本项目当前是单轨模型：

- 飞书只负责展示应用入口
- 服务端只负责鉴权和发卡片
- 影刀负责表单收集与提交触发

文档入口：

1. [飞书端实施手册](./feishu-setup.md)
2. [影刀端实施手册](./yingdao-setup.md)
3. [服务端实施手册](./service-setup.md)
4. [多维表格模板与字段说明](./bitable-template.md)
5. [CSV 模板文件](./bitable-csv/)
6. [Docker 部署说明](./Docker.md)

建议顺序：

1. 先在影刀侧为每个应用准备分享表单
2. 再在飞书侧配置菜单、事件订阅和多维表格
3. 最后部署服务端并做联调

联调通过标准：

- 菜单按钮可以返回“可用影刀应用”卡片
- 不同用户只能看到自己有权限的应用
- 点击应用按钮后直接打开对应影刀分享表单
